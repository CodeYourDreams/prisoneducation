So far, we've just learned how to grab user information with forms on Flask, but we don't know how to store that information long term.

In this lesson, we will store learn how to store user-inputted information by linking our Flask Applications to a database. We will do this with SQL, structured query language, which will allow you to store data in a format similar to a large Excel sheet.

In a lot of cases, it would be beneficial to learn SQL syntax and statements to work with SQL databases, but we can use libraries that allow us to just use Python code, so that we don't have to learn a whole new language.

We will use an ORM, object rotational mapper, which can directly use python instead of SQL syntax to do anything we might need, such as create or updating from our database. The most common ORM for python is SQLAlchemy, which translates Python classes to tables and is one way to interact with a database

We know already how to make a table in pure Python, so now lets look at how we would do it using SQLAlchemy.


Breaking down SQLAlchemy

  • Before creating a table in SQLAlchemy, you must create a Base

  • declarative_base() is a function that creates a base for you!

    • Bases make it easy to create columns in your table

  • Your class will inherit from the base that declarative_base() returns


  • The next step is to give your table a name !

    • Make sure to name your table something descriptive and memorable.

  • Then, we can define the columns in our table

    • These correspond to the attributes in our class

  • One column must have primary_key = True

  • Pick the variable names for these columns carefully - they become your column headers!
