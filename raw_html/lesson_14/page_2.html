HTTP methods: GET vs POST

In the browser we have 2 main requests : GET and POST.


GET

POST

  • Used for retrieving data

  • Parameters are sent in the URL

  • Can only send text data

  • All data is clearly visible – not good for passwords

  • Used for sending data

  • Parameters are sent in the request body

  • Can send any kind of data

  • Usually used for passwords


In other words ,

GET - is when you put in a link/route in the url bar, and GET an HTML page (a view) in return!


POST - is when you are already on that HTML page you just got , and you give the page information!

→ Such as login info.

→ Or filling a form.
