A web application isn't very helpful if a user can only go to the index ( / ). We can add more routes by using @app.route() . The syntax for @app.route() is: 

@app.route(<URL>)\n

Where <URL> is the route portion of a URL. For example, if you want to add a function to your app to display a message when a user goes to localhost:5000/hello , you would add the second route in the code above to app.py

Note the slash at the front of /hello . In another example, if our Flask website is http://myapp.com , and our app.py includes this function: 

@app.route('/example_page')\ndef example_page():\n    return '<p>This is an example</p>'\n

Then the URL for the function example_page() is http://myapp.com/example_page . However , during testing, the URL will be http://localhost:5000/example_page .

If you try going to http://localhost:5000/chicago , you'll see you get a 404 error . This error means that the page does not exist.
