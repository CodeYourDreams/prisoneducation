What is User Testing?

Definition: the process of collecting data from the user, qualitative and quantitative ones, to improve user experience


Usability: the efficiency and ease of use of a website by a user in order to satisfy the user’s needs.

How to measure:

  • Eye-tracking

  • Heat maps

  • Counting clicks

  • Time spent on each page

  • User Journey

  • Surveys


Testing may look a little different depending on whose experience you are looking at. Here are a few examples of what the process could look like.

TESTING EXPERIENCE: PERSON 1 (WEBSITE DESIGNER)

  • Decide what User Group you want your tester to be from

  • Run your website

  • Have a pencil and paper (or sticker note on your computer) open and ready

    • If the User runs into issues, you want to be able to write down when / why those issues occurred!


TESTING EXPERIENCE: PERSON 2 (USER)

  • Pretend to be in the User Group specified

  • Use the website

    • Normally in person, or via a website online

  • Keep notes

TESTING EXPERIENCE: FEEDBACK

“I like, I wish, I wonder”

  • “I like” is a good thing

  • “I wish” is something you want the app to have

  • “I wonder” is something you question or an idea you have to make it better
